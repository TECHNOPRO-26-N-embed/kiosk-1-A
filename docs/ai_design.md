# 基本設計書
### 3.1 画面遷移図

> テキスト形式で画面の状態遷移を記述します。

メインメニュー → [商品選択] → 商品詳細画面 → [購入] → 購入確認画面 → [確定] → 完了画面 → メインメニュー
　　　　　　　　　　↓
　　　　　　　[管理者モード] → 管理者メニュー → [在庫管理/売上確認/ログ閲覧] → 管理者メニュー → メインメニュー

### データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 商品ID | 各商品を一意に識別する番号 | 001, 002, ... |
| 商品名 | 商品の名称 | コーラ、緑茶、コーヒーなど |
| 在庫数 | 各商品の現在の在庫数 | 25 |
| 価格 | 各商品の販売価格 | 120 |
| 売上履歴 | 購入日時・商品・金額などの記録 | 2026/05/19, コーラ, 120 |
| 操作ログ | 操作内容や日時の記録 | 2026/05/19, 在庫補充 |
| 季節区分 | 商品の季節ごとの分類 | 春、夏、秋、冬 |
| 管理者パスワード | 管理者モード用の認証情報 | "admin123" |
| 投入金額 | お客さんが投入した金額 | 500 |
| つり銭 | 返却すべき金額 | 380 |

### CSVファイル保存形式の定義

#### 売上記録CSV（例: sales.csv）
| 項目名 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 日時 | 購入日時（YYYY/MM/DD HH:MM:SS） | 2026/05/19 10:23:45 |
| 商品ID | 購入された商品のID | 001 |
| 商品名 | 購入された商品の名称 | コーラ |
| 金額 | 1回の購入金額 | 120 |
| 購入数 | 購入された個数 | 1 |
| 残在庫数 | 購入後の在庫数 | 24 |

保存例：
2026/05/19 10:23:45,001,コーラ,120,1,24

#### 操作ログCSV（例: operation_log.csv）
| 項目名 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 日時 | 操作日時（YYYY/MM/DD HH:MM:SS） | 2026/05/19 09:00:00 |
| 操作内容 | 実施した操作の内容 | 在庫補充 |
| 担当者ID | 操作を行った管理者や利用者のID（任意） | admin01 |

保存例：
2026/05/19 09:00:00,在庫補充,admin01
